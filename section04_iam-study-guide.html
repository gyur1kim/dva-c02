시험 대비 핵심 개념 정리

IAM 개요

Identity and Access Management

AWS 계정의 인증(Auth·N)과 인가(Auth·Z) 관리

글로벌 서비스 — 리전에 종속되지 않음

처음 생성 계정 = 루트 계정 (Root Account)

구성요소

Users — 실제 사람이나 서비스를 나타냄

Groups — 사용자의 집합 (다른 그룹 포함 불가!)

Roles — AWS 서비스에 임시 권한 부여

Policies — 권한을 JSON 형식으로 정의

Policy 구조

Version 문서 레벨 — Statement에 포함 ✗

Statement (Effect, Action, Resource,

Principal, Condition, Sid)

종류: AWS Managed · Customer Managed · Inline

고객 책임 — Security IN the Cloud

2

IAM 정책의 Statement(문)을 구성하지 않는 요소는?

정책 구조

정답: Version (버전)

Version은 정책 문서(document) 레벨의 요소입니다. 개별 Statement에 포함되지 않습니다.

Statement 요소: Effect · Action · Resource · Principal · Condition · Sid(선택)

3

루트 계정 보안을 높이려면 어떻게 해야 할까요?

보안

MFA (다단계 인증) 활성화 — 핵심 정답

• 루트 계정에 MFA 반드시 설정

• 일상 업무에 루트 계정 사용 금지

• 루트 자격 증명 절대 공유 금지

• 루트 액세스 키 생성 자제

4

다음 중 IAM 정책의 종류는?

정책 종류

AWS Managed Policy — AWS가 미리 생성·관리

Customer Managed Policy — 고객이 직접 생성·관리

Inline Policy — 특정 사용자/그룹/역할에 1:1 직접 연결

모두 JSON 형식으로 작성됩니다.

5

다음 중 IAM 모범 사례(Best Practice)는?

모범 사례

• 루트 계정 MFA 활성화

• 루트 계정 일상 사용 금지

• 사용자별 개별 IAM 계정 생성 (공유 금지)

• 그룹을 통한 권한 관리 (사용자에 직접 정책 부여 지양)

• 최소 권한 원칙 적용

• 액세스 키 주기적 교체 및 불필요 키 삭제

6

IAM 사용자와 관련하여 잘못된 것은?

사용자

✗ 잘못된 것: "IAM 사용자는 반드시 하나의 그룹에만 속해야 한다"

올바른 사실:

• 사용자는 여러 그룹에 동시에 속할 수 있음

• 사용자는 어떤 그룹에도 속하지 않을 수 있음

• 사용자에게 직접 정책을 부여할 수도 있음

7

다음 중 IAM 보안 도구는 무엇일까요?

보안 도구

Credentials Report (자격 증명 보고서)

계정 레벨 — 모든 사용자 자격 증명 현황을 CSV로 다운로드

Access Advisor (액세스 관리자)

사용자 레벨 — 서비스별 마지막 접근 시간, 불필요 권한 식별

8

IAM 사용자 그룹에는 IAM 사용자 및 기타 사용자 그룹이 포함될 수 있다. (True / False)

O/X 문제

✗ False

IAM 그룹에는 IAM 사용자만 포함 가능합니다.

다른 그룹(중첩 그룹)은 포함할 수 없습니다.

그룹 → 사용자 ✅

그룹 → 그룹   ✗

9

IAM 권한과 관련하여 어떤 원칙을 적용해야 할까요?

원칙

최소 권한 원칙 (Principle of Least Privilege)

• 작업 수행에 필요한 최소한의 권한만 부여

• 불필요하게 광범위한 권한 부여 금지

• 필요에 따라 점진적으로 권한 추가

Access Advisor로 실제 사용 권한 검토 후 정리

10

IAM 역할(Role)의 올바른 설명은?

역할

AWS 서비스나 외부 엔티티에 임시 권한을 부여하는 메커니즘

• EC2 인스턴스, Lambda 함수 등 AWS 서비스에 권한 부여

• 영구 자격 증명 없이 임시 보안 자격 증명(STS) 발급

• 예: EC2 인스턴스가 S3에 접근할 때 역할 사용

Cross-account 접근도 역할(Role)로 구현

핵심 기억사항

루트 계정은 초기 설정 후 MFA 즉시 활성화, 이후 사용 자제
IAM 그룹에는 사용자만 포함 가능 — 다른 그룹(중첩) 포함 불가
Version은 정책 문서 레벨 요소, 개별 Statement에 포함 안 됨
IAM은 글로벌 서비스 — 리전 무관
사용자는 여러 그룹 소속 가능 / 그룹 미소속도 가능

IAM 보안 도구

Credentials Report — 계정 전체 사용자 자격 증명 현황 CSV
Access Advisor — 사용자별 서비스 접근 이력, 마지막 사용 시간

공동 책임 모델

AWS 책임: 인프라, 데이터 센터, IAM 서비스 운영
고객 책임: IAM 사용자 관리, 정책 설정, MFA, 키 교체

정책 & 역할 핵심

정책 종류: AWS Managed · Customer Managed · Inline (모두 JSON)
역할은 AWS 서비스에 임시 자격 증명(STS)을 통해 권한 부여
